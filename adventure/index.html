JSON Adventure Map Layout

Action Layout

Overrides for responses to the verbs when acted upon the object or in the room.
Example:

123actions: {
  look: 'This is the description of the room or object when inspected'
}

Possible Actions

  • name
    • Default: Unnamed Room
    • Alias(es): none
    • This returns the name of the room.
  • look
    • Default: There is nothing special about the [$object]
    • Alias(es): l, examine, x
    • When called as l or look, it takes no parameters and gives a general description of the room following the look action in the current room.
    • When called as x or examine, it takes one parameter, the object to examine, and gives a detailed explanation of the object following the look action in the given object.
  • take ($object)
    • Default: You have picked up the [$object][Allow]
    • Alias(es): get, pick, pick up
    • This attempts to take an object and place it in the user’s inventory.
  • drop ($object)
    • Default: You have dropped the [$object][Allow]
    • Alias(es): place (on floor/ground), throw
    • This attempts to drop an object on the ground from the user’s inventory. The object will now have it’s obvious property set to true if it was undefined.
  • place ($object) in/on ($target)
    • Default: [$target==?You have placed the [$object] in the [$target][Allow]:You cannot place the [$object] in the [$target]]
    • Alias(es): insert in/on, put in/on, drop in/on, pour in/on, throw in/on/at
    • Attempt to insert an object inside another object. Only called on the container in the placement.
  • remove ($object) from ($target)
    • Default: [$target==?You have taken the [$object] from the [$target][Allow]:You cannot remove the [$object] from the [$target]]
    • Alias(es): take (when inside container), get
    • Attempt to remove an object from a container. Only called on the container.
  • Other:
    • You can create custom commands by adding an undefined action to the list of actions for a room or an object and any subjects and direct objects will be passed.
  • enter / leave
    • Not a real command, defines something to say on entry or exit of a room

Map Layout

1234567891011121314151617181920212223map: {
  actions: {
    start: '_unique_room_name_',
    // See actions above
    _verb_: '_response_'
  },
  rooms: {
    _unique_room_name_: {
      actions: {...},
      objects: {
        _unique_object_name_: {
          // Actions...
        },
        _other_object_name_: {
          // Actions...
        }
      },
      _other_room_name_: {
        ...
      }
    }
  }
}

Each room must have a unique identifiable name to be used as a reference when directing the player to a room, and each object must have a unique identifiable name to be used as a reference when doing something to the object.

If the key obvious is set to true, the object will be obviously mentioned when looking in a room. Any objects that have been placed will become obvious.

Response Formatting

The string responses can be formatted to achieve dynamic results. For example,
Bye![->some_room_name] will direct the player to a room after printing Bye. These [...] statements can be interspersed within the string wherever normal text would be accepted. Whitespace can optionally be inserted between significant characters as shown.

Special Formats

  • [-> some_room_name ] will transport a player to a room specified with the name given (often used for movement: north: '[->northward_room]')
  • [ var_name ] will print the value of the variable. A variable name that starts with a $ is a special parameter reserved for $subject and $object, subject of the verb and direct object of the verb respectively. A variable name that starts with a @ is a truthy value of whether the object name exists in the player’s inventory. Can also find properties of objects with obj_name.prop_name to allow you to set the obvious property.
  • [ var_name == value ? Yay it's true! : Nope. It's false. ] is a conditional statement which prints the first string (optionally contianing more [...]) if true and the second string if false.
  • [ var_name = value ] sets the value of var_name to a value. Only works on non-special variables (not $ or @)
  • [Allow/Deny] Allows or denies the attempted action. The most specific is acknowledged. For example, if allowed in the map’s action list, but denied in the room’s action list, then it is denied.

Example Map

1234567891011121314151617181920212223242526272829303132333435363738394041424344454647484950map: {
  actions: {
    _default: 'Sorry, I do not understand how to ${verb}.'
  },
  rooms: {
    example_room: {
      actions: {
        name: 'Example Room',
        look: 'This is an Example Room. [example_looked?You have seen this [found_gold_key?room:rock] before...:[example_looked=True]] There is a room to the north.',
        help: 'I refuse to help you.',
        north: '[->next_room]'
      },
      objects: {
        example_object: {
          name: 'Example Container',
          obvious: true,
          take: 'Sorry. This object cannot be lifted.[prevent]',
          add: '[$object==trap_object?You are very smart... You may leave now[allow]:This is a very picky container[prevent]]'
        },
        trap_object: {
          name: 'Special Object',
          take: 'You are doomed.',
          obvious: true,
          drop: 'Sorry. You cannot drop this object on the floor. :P[prevent]'
        },
        gold_key: {
          name: "[@gold_key?A Golden Key:A rock]",
          take: 'You have found a Golden Key![found_gold_key=True]'
        }
      }
    },
    next_room: {
      actions: {
        name: 'Other Room',
        look: 'There is a locked door at the end of the hall.',
        south: '[->example_room]'
      },
      objects: {
        door: {
          name: "Antique Door",
          look: "An old antique door with a lock. It looks very important."
        },
        lock: {
          name: "Antique Lock",
          add: '[$object==gold_key?The key fits![allow][win]:Good luck with that...[prevent]]'
        }
      }
    }
  }
}
